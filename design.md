# 济事楼导览系统设计方案

## 2. 项目背景与目标
### 2.1 背景
高校教学楼空间复杂、访客与新生对教室/办公室位置不熟悉，传统楼层平面图难以快速定位。需要一个可交互的室内导览界面，支持快速发现与理解空间信息。

### 2.2 目标
- 降低用户识别与定位房间的成本
- 通过“地图-详情”联动提升信息获取效率
- 使用流式导航（fluid navigation）维持空间连续感，避免认知断裂

## 3. 目标用户与使用场景
### 3.1 目标用户
- 新生、访客
- 教师与行政人员
- 校内日常使用者

### 3.2 典型场景
- 查找某位老师办公室
- 快速定位某教室
- 在楼层切换中比较空间分布

## 4. 设计原则
### 4.1 以空间为中心
界面结构围绕地图组织，突出空间位置与方向感。

### 4.2 低负担探索
通过 hover 提示与渐进式细节（tooltip -> 详情）减少操作成本。

### 4.3 流式导航（Fluid Navigation）
保持交互前后界面元素的连续性，通过平滑过渡与位移让用户感知“仍在同一空间内”。

## 5. 信息架构
- 顶层：楼层选择、搜索入口
- 核心：地图区域（PNG 视觉层 + SVG 热点层）
- 辅助：详情面板（图片、人员、房间信息）

信息呈现遵循“先空间位置、后详细内容”的顺序。

## 6. 交互与布局设计
### 6.1 总体布局
- 左侧为主地图区域，右侧为可展开详情面板
- 顶部或侧边提供楼层切换与搜索

### 6.2 交互流程
1. 用户在地图上 hover 热点
2. Tooltip 提示房间信息
3. 点击房间
4. 地图平移缩放，详情面板展开
5. 用户查看详细信息或拖拽分隔条调整比例

### 6.3 流式导航原则的应用与理由
- **空间连续性**：点击房间后，地图并未被替换为新页面，而是平移缩放并让详情面板“从同一界面中生长出来”。这种连续变换减少了视线跳跃，使用户保持对空间位置的心理模型。
- **注意力引导**：通过地图向左缩放与详情栏展开，注意力自然从位置过渡到内容。相比弹窗或跳转页，更符合探索式任务。
- **状态可追溯**：用户仍能看到原地图，随时回到空间视角，避免在内容详情中迷失。
- **操作成本降低**：连续的动画过渡让用户理解“点击导致细节展开”，不需要重新适应新界面。

因此，该布局与交互方式选择了“地图主视图 + 详情侧栏”的结构，并使用过渡动画保持流式导航体验。

## 7. 功能设计
### 7.1 地图展示
- PNG 作为视觉底图
- SVG 热点层与房间数据绑定

### 7.2 热点交互
- hover：高亮与 tooltip
- click：触发详情面板与地图变换

### 7.3 搜索
- 输入关键字
- 结果列表定位至对应房间

### 7.4 楼层切换
- 当前支持 4F
- 5F 作为扩展预留

### 7.5 详情面板
- 图片轮播与放大
- 房间信息、人员列表
- 支持拖拽调整宽度

## 8. 视觉与动效设计
- 轻量阴影与层级体现“信息上浮”
- 过渡动画保证空间感连贯
- 热点高亮与缩放用于强化可交互性

## 9. 数据与组件设计
### 9.1 数据模型
- Room：名称、编号、人员、图片等
- Staff：姓名、职位、联系方式

### 9.2 组件划分
- FloorMap：地图渲染与热点绑定
- RoomTooltip：悬浮提示
- RoomDetailPanel：详情信息
- SearchBar：搜索
- FloorSelector：楼层切换

组件之间通过状态中枢协调，保证交互一致。

## 10. 技术方案
- Vue 3 + Composition API
- TypeScript 数据类型约束
- Vite 构建
- CSS 过渡实现动画
